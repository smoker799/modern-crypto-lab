Euler Problem 182 - RSA 未隐藏消息分析

发布日期：2026-06-10

引言

RSA 作为最经典的非对称加密算法，其安全性依赖于大整数分解的困难性，但在密钥选择阶段仍存在一些容易被忽略的细节问题。Project Euler Problem 182 就聚焦于 RSA 中的“未隐藏消息”问题，这是一类加密后数值不变的明文，会直接泄露信息。本文将完整分析这个问题，并给出可直接运行的实现代码。

问题描述

给定两个质数 p=1009 和 q=3643，我们需要找出所有满足 1 < e < φ(n) 且 gcd(e, φ(n)) = 1 的公钥指数 e，使得未隐藏消息的数量最少，并求这些 e 的和。

未隐藏消息的定义：满足 m^e ≡ m mod n 的明文 m（0 ≤ m < n）。这类消息在 RSA 加密后不会发生变化，会直接泄露明文内容，因此需要控制其数量。

原理分析

根据数论和中国剩余定理，m^e ≡ m mod n 等价于同时满足：

  • m^e ≡ m mod p
  • m^e ≡ m mod q

因此，模 n 下解的数量等于模 p 下解数与模 q 下解数的乘积。

进一步分析，模质数 p 下 m^e ≡ m mod p 的解数为：

解数 = gcd(e-1, p-1) + 1

同理，模 q 下的解数为：

解数 = gcd(e-1, q-1) + 1

因此，n = p*q 下未隐藏消息的总数公式为：

count = (gcd(e-1, p-1) + 1) × (gcd(e-1, q-1) + 1)

我们的目标就是遍历所有合法的 e，找到使 count 最小的 e 并求和。

实现代码

import math

def count_unconcealed(e, p, q):
    """计算给定e对应的未隐藏消息数量"""
    g1 = math.gcd(e - 1, p - 1)
    g2 = math.gcd(e - 1, q - 1)
    return (g1 + 1) * (g2 + 1)

def solve_euler_182():
    p = 1009
    q = 3643
    phi = (p - 1) * (q - 1)
    min_count = float('inf')
    valid_es = []

    # 遍历所有满足条件的e
    for e in range(2, phi):
        if math.gcd(e, phi) != 1:
            continue
        cnt = count_unconcealed(e, p, q)
        if cnt < min_count:
            min_count = cnt
            valid_es = [e]
        elif cnt == min_count:
            valid_es.append(e)

    print(f"最少未隐藏消息数: {min_count}")
    print(f"满足条件的e的数量: {len(valid_es)}")
    print(f"所有e的和: {sum(valid_es)}")

if __name__ == "__main__":
    solve_euler_182()

运行结果与结论

  • 最少未隐藏消息数：9
  • 满足条件的 e 的和：6730666

从结果可以看出，即使 e 与 φ(n) 互质，也会存在未隐藏消息。选择合适的 e 可以显著减少这类消息的数量，提升 RSA 的安全性。这也提醒我们，在实际应用中选择 RSA 公钥指数时，不仅要保证与 φ(n) 互质，还要考虑这类安全细节。
