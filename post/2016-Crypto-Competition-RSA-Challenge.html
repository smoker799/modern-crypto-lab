一、赛题背景与问题描述

RSA 是目前应用最广泛的公钥密码体制，其安全性依赖于大整数素因子分解的数学困难性。本次赛题针对一款自定义分片+比特填充的 RSA 加密软件进行破译，目标：

  1. 还原 RSA 全套密钥参数：素数 p、q、模数 N、加密指数 e、解密指数 d
  2. 截获密文并逆向还原原始通关明文
  3. 分析该定制 RSA 方案的安全缺陷

1.1 标准 RSA 算法流程

1）密钥生成

  1. 选取两个大素数 p、q，计算模数 N = p × q
  2. 计算欧拉函数 φ(N) = (p-1)(q-1)
  3. 选取加密指数 e，满足 gcd(e, φ(N)) = 1
  4. 求解 e 模 φ(N) 的逆元 d，满足 e × d ≡ 1 (mod φ(N))
  5. 公钥 (e, N) 公开，私钥 (d, p, q) 保密

2）加解密公式

  • 加密：c = m^e mod N
  • 解密：m = c^d mod N

1.2 赛题自定义加密规则（核心约束）

该软件并非原生 RSA，增加了明文分片 + 512 比特固定填充规则：

  1. 模数 N 为 1024 比特，p、q 为大素数；
  2. 单次加密最多 8 个 ASCII 字符，超长明文按 8 字符分片；
  3. 512bit 消息填充格式（高位→低位）：
    • 64 比特标志位
    • 32 比特通信序号
    • 若干补零比特
    • 最低 64 比特：8 个分片字符对应的 ASCII 十六进制数据（真实明文）
  4. 传输帧格式：1024bit N | 1024bit e | 1024bit 密文 c，全部采用十六进制存储；
  5. 加密指数统一使用 e = 0x10001（65537）。

三、破译思路与实验步骤

3.1 整体破译流程

  1. 提取帧数据中的 N、e、c；
  2. 利用私钥 d 执行 RSA 解密：m = c^d mod N，得到完整 512bit 填充明文；
  3. 根据填充规则，截取最低 64 比特十六进制串；
  4. 将十六进制转为 ASCII 字符，得到单分片明文；
  5. 拼接所有分片，还原完整通关密语；
  6. 反向验证密钥生成、加密流程，核对所有参数。

3.2 核心操作要点

  1. 十六进制字符串与大整数互相转换；
  2. Python 原生 pow(a,b,mod) 实现高效大数模幂（RSA 加解密核心）；
  3. 按比特位截取有效明文段，剥离标志位、通信序号、补零位；
  4. 验证 gcd(e, φ(N))=1、模逆元 d 的正确性。

四、运行结果

  1. 分片解密结果：依次解密 4 组密文，得到 4 段 8 字符分片；
  2. 拼接后完整明文： 
    This is a test of my RSA system.
  3. 所有密钥参数、加解密运算均与赛题原始数据完全吻合，破译成功。

五、安全漏洞分析

结合本次定制 RSA 方案，总结存在的安全风险：

  1. 固定填充格式漏洞
    填充位结构完全固定（64位标志+32位序号+补零），攻击者可直接定位真实明文所在比特段，极易逆向。
  2. 低加密指数风险
    e=0x10001 为行业常用小指数，虽运算快，但易遭受低加密指数攻击。
  3. 明文分片重复风险
    软件支持重复发送相同分片，攻击者可利用重放攻击、明文-密文映射表进行分析。
  4. 素数选取不规范
    第二组密钥中素数 q 位数远小于 p，大幅降低大数分解难度，违背 1024bit RSA 安全设计原则。

补充：标准 1024bit RSA 依靠大整数分解保证安全，若无上述实现漏洞，单纯暴力分解模数在现有算力下难以完成。

六、实验总结

  1. 本次实验完整复现了分片+自定义填充的 RSA 加解密全过程，成功破译密文并还原全部密钥与明文。
  2. 掌握了 RSA 数论基础：欧拉函数、模逆元、大数模幂运算。
  3. 算法本身安全 ≠ 工程实现安全，上层分片、填充、素数选取、参数配置都会引入致命漏洞。
  4. 实际密码系统设计，不仅要选用安全算法，更要严格规范参数与协议设计。
